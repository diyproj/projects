Tool ecosystem: Cross-platform USB adapters for rapid development and real-time testing

Ready-made commercial product in a flash-drive form factor. Connect sensors directly to a PC without breadboards or extra cables

The platform was designed by an engineer for engineers and has passed extensive testing while debugging measurement and industrial equipment.
Who this product is for:
  • for embedded developers and QA engineers: fast testing of new chips/sensors (I2C, SPI, 1-Wire, UART, etc.). Testing hypotheses, debugging registers, and capturing telemetry in real time directly through a PC.
Generation of pulses / sequences with defined parameters, measuring time intervals. Power for connected devices +5V, +3.3V (+1.8V - on request).
  • for developers of automation systems (PC software): integrating hardware into PC software (Python, C#, C++) without writing MCU firmware. A ready-made hardware “layer”.
  • for users Arduino: moving from bulky development boards with a bunch of wires to a monolithic final device in a protective housing.
Features
  • Simultaneous independent bus operation: Separate physical pin groups for each interface. You can simultaneously poll an I2C clock, SPI memory, and a 1-Wire thermometer.
  • Choice of architecture for your tasks: The platform is supplied with the MCU of your choice: budget STM8S, classic ATmega8/328, or modern megaAVR 0-series (ATmega808).
  • Software for the PC has a built-in formula interpreter. The user enters a formula and gets the physical quantity (for example, degrees Celsius, hectopascals...) directly on screen, without manually converting bytes from the registers.
  • Deep software configuration (On request):
    • Mode VCP (Virtual COM port): For working with scripts and full compatibility with Arduino IDE.
    • Mode USBxpress (Silicon Labs Direct Driver): Bypassing the Windows COM port stack. Direct access via the API for high-speed Real-Time data streaming (accelerometers, vibration) without OS delays.
  • Speed of data transfer up to 500,000 bit/s for latency-critical tasks.
  • Choice of quartz resonator frequency (on request) for precise measurement / setting of time intervals.
  • +1.8V regulator (on request).- for testing chips powered at +1.8V without a level-shifter IC. Example chip: DDS AD9913.
  • An additional board can be placed in the housing on top of the main one. Example: various transistor-key boards for solenoids, stepper motors, 7-segment indicators.

Thermostat

The device was made for temperature control at a country house, and is used for preliminary temperature testing of electronic equipment in a lab, in a homemade thermostat - it's very convenient: only the sensor goes into the thermostat, all the controls are outside, the heater (a light bulb) turns off/on - you record the temperature and take readings from the equipment under test.

The previous device was built with a thermistor and a dual op-amp, but adjusting the desired temperature with a trimmer resistor, and selecting a resistor to provide hysteresis, was quite a hassle.

The presented controller is quite simple, built on an Attiny26 MCU and a 3-digit LED display, on which the temperature is shown in the format XX.X

Three buttons let you set the ON and OFF temperatures of the heater (one button selects the mode: <room temperature display> – <set TºC turn-on> – <set TºC turn-off>, the <+> button increases and the <-> button decreases the temperature value in the T setting modesºC).

The DS18B20 IC, used as the temperature sensor, is connected to the controller with just two wires several meters long. The DS18B20 is an IC with a serial interface in a TO-92 package. The temperature measurement error is less than ±0.5ºC in the range -10ºC…+85ºC, and ±2ºC in the range -55ºC…+125ºC without additional calibration.

The controller switches 220V mains voltage to a galvanically isolated load (heater), providing current up to 4A (at the country house, a gas igniter consuming less than 40W is used).

What's more interesting is the device's construction: all the components (the controller PCB with an indicator, opto-isolator, triac, regulator, and power transformer), except for the temperature sensor IC brought out on wires, are housed inside a mains power-supply enclosure with a mains plug; on the side of the enclosure is an outlet for connecting the load. The indicator and the button-access holes are on the back panel of the enclosure. All that's needed is to cut a window for the indicator, 3 holes for button access, and 3 slots inside the enclosure using a heated soldering iron.

Two-temperature meter

The device was made for a diesel car (Volkswagen); when starting, the customer needs to know the outside temperature, while the interior temperature is just nice to have. Two temperature sensors and two 3-digit temperature indicators are used, plus 2 LEDs to indicate a "-" sign.

The indicator board is 46 x 31mm, the microcontroller board is 42 x 36mm (per the customer's requirements for fitting the dashboard space), powered from the car battery, 12V-15V (power from +5V is also possible).

BinSoft: an application for ECU tuning

The BinSoft application provides tuning of fuel-injected engines by correcting the software of automotive onboard computers. The program lets you change the parameters of Electronic Engine Control Systems (ECU), which are stored in the software of the car's onboard computer (in the ROM).
The software can be read from the ROM ROM using the programmer, saved as a file, the file is corrected with the BinSoft program, and after saving it is written back into the ROM ROM using the programmer.
. The type of the original software is detected automatically, including for software that has already been corrected before.
Software is corrected for various modifications of the ECU::  Bosch, Yanvar, VS5

Balancing controller for rotating objects

A controller for balancing rotating objects, such as motor rotors. Balancing uses a stand with soft suspension and an accelerometer, as well as a stroboscope disk with angular marks to determine the direction of imbalance. The disk rotates together with the rotor. The object being balanced is spun up to 2000...3000 RPM; the signal produced at the output of the accelerometer is proportional to the magnitude of vibration acceleration. This signal, arising from the imbalance of the rotating object, is amplified and fed to the controller. The controller determines the signal frequency (the rotor's rotation frequency), and the signal amplitude, proportional to the vibration acceleration measured by the accelerometer, is converted into vibration displacement, proportional to the object's unbalanced mass. The direction of imbalance on the disk with angular marks is indicated by a pulsed LED backlight controlled by the controller. The indicator displays the vibration displacement amplitude; pressing the "Frequency" button displays the rotation frequency. A balancing mass, proportional to the vibration displacement, is added in the direction opposite to the direction of imbalance. Although the direction of mass imbalance corresponds to the maximum signal value (this direction is offset by 90 degrees relative to the zero crossing of the signal proportional to vibration acceleration), the controller provides the ability to change the angular offset of the stroboscope disk's backlight from 0 to 359 degrees in 1-degree steps relative to the zero-degree point (the signal's zero crossing). The offset angle is adjusted with buttons and stored in the controller for a specific stand. In this way, instead of highlighting the direction of imbalance, the disk can be set to highlight the direction in which the balancing weight should be added. The controller can be reprogrammed to display the imbalance angle without a stroboscope. In this case a sensor or a mark (optical, magnetic, ...) will be needed instead of the stroboscope disk.

8-channel weighing controller

Of course, the device can be used not only with load cells intended for weight measurement, but also with other sensors. Signals from the sensors can be either unipolar or bipolar; the maximum ADC input voltages corresponding to full scale can be selected from the range 20mV, 40mV, 80mV, 160mV, 320mV, 640mV, 1.28V, 2.56V. The ADC uses digital filtering that reduces 50Hz interference.

Distinctive features of the developed device

    the use of a separate delta-sigma ADC for each connected load cell,

    the number of connected sensors is selected by the user (from 1 to 8),

    data transfer from each ADC to the display unit - via RS485 over a distance of about 100m,

    the ability to linearize load-cell characteristics at several points (here, 4 points); for this, a keypad (13 keys: 10 digits, decimal point, ENTER, CANCEL) is connected to the display unit during calibration; a reference weight 1 is placed on the sensor being calibrated, its weight is entered from the keypad, and this is repeated for weights 2, 3, 4; based on the set weights and entered values, the coefficients used to display weight in normal operating mode are determined automatically,

    sensors of different types with different parameters can be connected to the display simultaneously; the display shows the sum of the signals from each sensor, each multiplied by its own individual coefficient.
